> This checklist maps ACLS mitigations against the OWASP Top 10 (2021). Every item must be reviewed before each production release.

---

## OWASP Top 10 — 2021

### A01: Broken Access Control

| Control | ACLS Mitigation | Status |
|---|---|---|
| Enforce access control server-side | `[Authorize(Roles)]` on all controllers. `TenancyMiddleware` enforces `PropertyId` scoping. | ✅ |
| Deny by default | All endpoints require authentication except `POST /auth/login` and `POST /auth/register`. | ✅ |
| Log access control failures | `AuditLog` records all actions. 403/404 responses logged at Warning level. | ✅ |
| Rate limit API access | Not implemented in V1. Planned for V2. | ⚠️ V2 |
| Invalidate JWTs on server-side | Not implemented in V1 — token lifetime is 60 minutes. Token revocation planned for V2. | ⚠️ V2 |

### A02: Cryptographic Failures

| Control | ACLS Mitigation | Status |
|---|---|---|
| Encrypt data in transit | HTTPS/TLS enforced in production (NFR-04). | ✅ |
| Do not store sensitive data unnecessarily | Only `PasswordHash` (BCrypt) stored. No credit card data in scope. | ✅ |
| Use strong, up-to-date algorithms | HMAC-SHA256 for JWT. BCrypt work factor ≥12 for passwords. | ✅ |
| Never commit secrets | All secrets in Key Vault or environment variables. `appsettings.Development.json` gitignored. | ✅ |

### A03: Injection

| Control | ACLS Mitigation | Status |
|---|---|---|
| Use parameterised queries | EF Core LINQ queries — no raw SQL in V1. | ✅ |
| Validate and sanitise input | FluentValidation on all command inputs. Max-length constraints on all string fields. | ✅ |
| Use ORM properly | `DbContext` only in `ACLS.Persistence`. No string concatenation into queries. | ✅ |

### A04: Insecure Design

| Control | ACLS Mitigation | Status |
|---|---|---|
| Threat model documented | This document + `docs/10_Security/threat_model.md`. | ✅ |
| Domain logic separated from infrastructure | Clean Architecture enforces this structurally. | ✅ |
| Business logic on the server | `ai_collaboration_rules.md` Rule 1 — thin frontends. | ✅ |

### A05: Security Misconfiguration

| Control | ACLS Mitigation | Status |
|---|---|---|
| No default credentials | No default admin accounts. First Manager account created via seed script with a documented temporary password that must be changed. | ✅ |
| Disable unnecessary features | Swagger UI disabled in production (`builder.Environment.IsDevelopment()` guard). | ✅ |
| Error messages don't expose internals | `ExceptionHandlingMiddleware` returns generic `System.InternalError` for 500s. Stack traces never serialised. | ✅ |
| Security headers | HSTS enforced by Azure App Service. Additional headers (X-Content-Type-Options, X-Frame-Options) set in middleware. | ✅ |

### A06: Vulnerable and Outdated Components

| Control | ACLS Mitigation | Status |
|---|---|---|
| Keep dependencies up to date | `tools/scripts/update-dependencies.ps1` — run before each release. | ✅ |
| Monitor for CVEs | GitHub Dependabot alerts enabled on repository. | ✅ |

### A07: Identification and Authentication Failures

| Control | ACLS Mitigation | Status |
|---|---|---|
| Strong password hashing | BCrypt work factor ≥12. | ✅ |
| JWT validation on every request | Signature, expiry, issuer, audience all validated. | ✅ |
| Brute force protection | Not implemented in V1. | ⚠️ V2 |
| Multi-factor authentication | Not implemented in V1. | ⚠️ V2 |

### A08: Software and Data Integrity Failures

| Control | ACLS Mitigation | Status |
|---|---|---|
| Verify integrity of data in transit | HTTPS ensures transport integrity. | ✅ |
| CI/CD pipeline security | GitHub Actions with branch protection. Secrets via GitHub Actions Secrets — not in workflow YAML. | ✅ |

### A09: Security Logging and Monitoring Failures

| Control | ACLS Mitigation | Status |
|---|---|---|
| Log authentication events | Login success/failure logged at Information/Warning level. | ✅ |
| Log access control failures | 403/404 responses logged. | ✅ |
| Immutable audit trail | `AuditLog` table — no update or delete operations. | ✅ |
| Alerts on anomalies | Azure Monitor alerts configured per `docs/07_Implementation/observability.md`. | ✅ |

### A10: Server-Side Request Forgery (SSRF)

| Control | ACLS Mitigation | Status |
|---|---|---|
| Validate all URLs before fetching | No user-supplied URLs are fetched by the server. Media URLs are generated by `IStorageService` — not user-supplied. | ✅ |

---

## Review Cadence

This checklist is reviewed:
- Before each production deployment
- After any new endpoint is added
- After any dependency update that affects authentication or data access
