── backend/ACLS.Application/ ───────────────────────────────────────────────────

  ACLS.Application.csproj
    Packages: MediatR 12.*, FluentValidation 11.*,
              FluentValidation.DependencyInjectionExtensions 11.*,
              Microsoft.Extensions.Logging.Abstractions 8.*,
              Microsoft.Extensions.DependencyInjection.Abstractions 8.*
    ProjectReferences: ACLS.Domain, ACLS.SharedKernel

  Common/Interfaces/ICurrentPropertyContext.cs
    int PropertyId { get; }
    int UserId { get; }
    Provides per-request property + user identity to handlers without coupling
    them to HttpContext.

  Common/Interfaces/IUnitOfWork.cs
    BeginTransactionAsync / CommitTransactionAsync / RollbackTransactionAsync
    Implemented by AclsDbContext (Persistence layer). Consumed only by
    TransactionBehaviour.

  Common/Behaviours/LoggingBehaviour.cs
    IPipelineBehaviour<TRequest, TResponse>
    Logs request name on entry; logs elapsed milliseconds on exit using
    Stopwatch.GetElapsedTime.

  Common/Behaviours/ValidationBehaviour.cs
    IPipelineBehaviour<TRequest, TResponse>
    Resolves all IValidator<TRequest> from DI; runs them concurrently via
    Task.WhenAll; throws ValidationException if any failures are found.
    Executes BEFORE the handler, not the handler's unit tests.

  Common/Behaviours/TransactionBehaviour.cs
    IPipelineBehaviour<TRequest, TResponse>
    Also defines the ITransactionalCommand marker interface (same file).
    Wraps only requests that implement ITransactionalCommand in a
    BeginTransactionAsync / CommitTransactionAsync / RollbackTransactionAsync
    block. Non-transactional commands pass through unchanged.

── Commands ─────────────────────────────────────────────────────────────────────

  Complaints/Commands/SubmitComplaint/
    SubmitComplaintCommand.cs
      IRequest<Result<ComplaintDto>>
      Nested record MediaUploadResult(string Url, string Type) for pre-uploaded
      blob references (no binary payload in command).
    SubmitComplaintCommandValidator.cs
      Rules: Title (3–200), Description (10–2000), Category (not-empty),
             Urgency (must parse to Urgency enum), UnitId (> 0),
             each MediaUrl must start with "https://".
    SubmitComplaintCommandHandler.cs
      Derives RequiredSkills from Category string.
      Calls Complaint.Create(...), persists via AddAsync, loops MediaUrls
      and calls AddMediaAsync for each, publishes ComplaintSubmittedEvent.

  Complaints/Commands/AssignComplaint/
    AssignComplaintCommand.cs
      IRequest<Result<ComplaintDto>>, ITransactionalCommand
    AssignComplaintCommandValidator.cs
      ComplaintId > 0, StaffMemberId > 0.
    AssignComplaintCommandHandler.cs
      Loads Complaint and StaffMember (both scoped to PropertyId).
      Calls complaint.Assign(staffMemberId) and staff.MarkBusy().
      Updates both aggregates, publishes ComplaintAssignedEvent.

  Complaints/Commands/ResolveComplaint/
    ResolveComplaintCommand.cs
      IRequest<Result<ComplaintDto>>, ITransactionalCommand
      Nested MediaUploadResult for completion photos.
    ResolveComplaintCommandValidator.cs
    ResolveComplaintCommandHandler.cs
      Calls complaint.Resolve() first — returns failure immediately if not
      IN_PROGRESS.  Loads assigned staff, calls staff.MarkAvailable(), updates
      both aggregates, attaches completion photos, publishes
      ComplaintResolvedEvent.

  Complaints/Commands/TriggerSos/
    TriggerSosCommand.cs, TriggerSosCommandValidator.cs,
    TriggerSosCommandHandler.cs
    Creates complaint, calls complaint.TriggerSos(), persists, publishes
    SosTriggeredEvent.

  Complaints/Commands/AddWorkNote/AddWorkNoteCommand.cs    (combined)
    Loads complaint, calls complaint.AddWorkNote(staffUserId, note), persists.

  Complaints/Commands/UpdateEta/UpdateEtaCommand.cs        (combined)
    Returns Result (no value). Calls complaint.UpdateEta(eta).

  Complaints/Commands/SubmitFeedback/SubmitFeedbackCommand.cs  (combined)
    Calls complaint.Close(rating, comment). Rating validated 1–5.

  Outages/Commands/DeclareOutage/
    DeclareOutageCommand.cs, DeclareOutageCommandValidator.cs,
    DeclareOutageCommandHandler.cs
    Calls Outage.Declare(...), persists, publishes OutageDeclaredEvent.
    No direct INotificationService call — notifications are dispatched by an
    event handler downstream (clean domain boundary).
