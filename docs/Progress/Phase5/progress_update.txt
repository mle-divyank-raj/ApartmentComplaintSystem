TASK 5.1 — NOTIFICATION SERVICE INFRASTRUCTURE
-----------------------------------------------

  backend/ACLS.Infrastructure/Notifications/INotificationChannel.cs
    public interface with ChannelType property and SendAsync method.
    NOTE: must be public (not internal) — NotificationService (public class) takes
    IEnumerable<INotificationChannel> in its constructor; internal would cause CS0051.

  backend/ACLS.Infrastructure/Notifications/EmailNotificationProvider.cs
    internal sealed class implementing INotificationChannel (Email channel).
    Reads ACLS_NOTIFICATION_KEY and ACLS_NOTIFICATION_EMAIL_FROM from IConfiguration.
    Throws InvalidOperationException on missing keys (fail-fast at startup).
    SendAsync: structured log stub — placeholder for SendGrid call.

  backend/ACLS.Infrastructure/Notifications/SmsNotificationProvider.cs
    internal sealed class implementing INotificationChannel (SMS channel).
    Reads ACLS_NOTIFICATION_KEY and ACLS_NOTIFICATION_SMS_FROM from IConfiguration.
    Throws InvalidOperationException on missing keys.
    SendAsync: structured log stub — placeholder for Twilio call.

  backend/ACLS.Infrastructure/Notifications/NotificationService.cs
    public sealed class implementing INotificationService.
    Constructor: IEnumerable<INotificationChannel>, IResidentRepository,
                 IUserRepository, ILogger<NotificationService>.
    BroadcastOutageAsync:
      GetAllByPropertyAsync() → build all send tasks across channels → Task.WhenAll()
    NotifyAllOnCallStaffAsync:
      iterate staff entities → look up User → fan-out across channels → Task.WhenAll()
    NotifyResidentAsync / NotifyStaffAsync:
      look up User by UserId → send via matching channel provider.

  backend/ACLS.Infrastructure/InfrastructureServiceCollectionExtensions.cs  (updated)
    Added DI registrations:
      services.AddScoped<INotificationChannel, EmailNotificationProvider>()
      services.AddScoped<INotificationChannel, SmsNotificationProvider>()
      services.AddScoped<INotificationService, NotificationService>()

TASK 5.2 — WORKER EVENT HANDLERS
---------------------------------
All handlers implement MediatR INotificationHandler<T>.
Each performs: notification dispatch → job execution → audit write.

  backend/ACLS.Worker/EventHandlers/ComplaintAssignedEventHandler.cs
    Trigger: ComplaintAssignedEvent
    Action:  NotifyStaffAsync(AssignedStaffMemberId, [Email, SMS])
             AuditEntry.Create(AuditAction.ComplaintAssigned, ...)

  backend/ACLS.Worker/EventHandlers/ComplaintResolvedEventHandler.cs
    Trigger: ComplaintResolvedEvent
    Action:  NotifyResidentAsync(ResidentId, [Email, InApp])
             CalculateTatJob.ExecuteAsync()
             UpdateAverageRatingJob.ExecuteAsync()
             AuditEntry.Create(AuditAction.ComplaintResolved, ...)

  backend/ACLS.Worker/EventHandlers/FeedbackSubmittedEventHandler.cs
    Trigger: FeedbackSubmittedEvent
    Action:  UpdateAverageRatingJob.ExecuteAsync(StaffMemberId, PropertyId)
             AuditEntry.Create(AuditAction.FeedbackSubmitted, ...)

  backend/ACLS.Worker/EventHandlers/OutageDeclaredEventHandler.cs
    Trigger: OutageDeclaredEvent
    Action:  BroadcastOutageNotificationJob.ExecuteAsync(OutageId, PropertyId)

  backend/ACLS.Worker/EventHandlers/SosTriggeredEventHandler.cs
    Trigger: SosTriggeredEvent
    Action:  IStaffRepository.GetAvailableAsync(PropertyId)
             IComplaintRepository.GetByIdAsync(ComplaintId, PropertyId)
             NotifyAllOnCallStaffAsync(availableStaff, complaint) — concurrent
             AuditEntry.Create(AuditAction.SosTriggered, ...)
    Note: injects IStaffRepository + IComplaintRepository directly because
          NotifyAllOnCallStaffAsync requires full domain entities, not IDs.

TASK 5.3 — WORKER BACKGROUND JOBS
-----------------------------------

  backend/ACLS.Worker/Jobs/CalculateTatJob.cs
    ExecuteAsync(int complaintId, int propertyId, CancellationToken)
    Formula: (decimal)(complaint.ResolvedAt.Value - complaint.CreatedAt).TotalMinutes
    Writes via: complaint.RecordTat(tatMinutes) → IComplaintRepository.UpdateAsync()
    Guard: early return if ResolvedAt is null.

  backend/ACLS.Worker/Jobs/UpdateAverageRatingJob.cs
    ExecuteAsync(int staffMemberId, int propertyId, CancellationToken)
    Fetches all complaints for staff member → filters ResidentRating.HasValue
    Formula: (decimal)ratings.Average()
    Writes via: staff.UpdateAverageRating(averageRating) → IStaffRepository.UpdateAsync()
    Guard: early return if no rated complaints.

  backend/ACLS.Worker/Jobs/BroadcastOutageNotificationJob.cs
    ExecuteAsync(int outageId, int propertyId, CancellationToken)
    Idempotent: early return if outage.NotificationSentAt.HasValue (already sent).
    Calls NotificationService.BroadcastOutageAsync() — Task.WhenAll() inside service.
    After broadcast: outage.MarkNotificationSent() → IOutageRepository.UpdateAsync()
    Audit: AuditEntry.Create(AuditAction.OutageDeclared, "Outage", outageId, propertyId)

BUGS RESOLVED
-------------

Bug 1 — CS0051: Accessibility inconsistency on INotificationChannel
  Symptom: INotificationChannel was declared internal but NotificationService (public)
           exposed it in a public constructor parameter → compiler error CS0051.
  Fix:     Changed INotificationChannel to public interface.

Bug 2 — CS0311: IDomainEvent does not implement MediatR.INotification
  Symptom: All 5 EventHandlers failed: "cannot use X as INotificationHandler<T> —
           no implicit reference conversion to MediatR.INotification".
  Root cause: IDomainEvent in SharedKernel had no relationship to MediatR's
              INotification. The existing Application handlers also had a latent
              runtime bug (Publish(object) casts to INotification internally).
  Fix:     Added PackageReference MediatR 12.* to ACLS.SharedKernel.csproj.
           Changed IDomainEvent to: public interface IDomainEvent : INotification.
  Impact:  All 5 Worker EventHandlers compile. Application-layer Publish() calls
           are now correct at runtime (no more silent cast failure risk).

Bug 3 — CS1061: UseAzureMonitor not found on OpenTelemetryBuilder in Worker SDK
  Symptom: Azure.Monitor.OpenTelemetry.AspNetCore was referenced; its UseAzureMonitor()
           extension method is unavailable in a Worker SDK project context.
  Fix:     Replaced package with Azure.Monitor.OpenTelemetry.Exporter.
           Updated Program.cs to use individual exporter methods:
             AddAzureMonitorTraceExporter()
             AddAzureMonitorMetricExporter()
             AddAzureMonitorLogExporter()
